使用 Markdown 发布

保持工作流简单。用 Markdown 撰写、提交到代码仓库，剩下的部署交给 GitHub Pages（或任意静态托管服务）。文件顶部的 Front Matter 将用于驱动搜索、筛选和文章元数据。

1. 创建 Markdown 文件

使用描述性文件名，例如 incident-response.md。

2. 添加有用的元数据

包括标题、描述、标签和分类，站点即可自动组织内容。

3. 提交并推送

当文件进入默认分支后，知识库会自动更新。

常见问题

  • 文章存放在哪里？ 所有 Markdown 文件都位于仓库的 docs/ 目录。
  • 如何分享？ 将你的自定义域名指向 GitHub Pages，或直接分享 https://yxuan18.github.io。
  • 可以嵌套内容吗？ 可以。将 Markdown 文件放入 docs/ 的子文件夹中，系统同样会索引。
  • 可以使用其他托管吗？ 当然。任何能够托管此仓库的静态站点服务都可以。
