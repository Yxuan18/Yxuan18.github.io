Publish with Markdown

Keep your workflow simple. Write in Markdown, commit to the repository, and GitHub Pages (or any static host) takes care of the rest. Front matter at the top of each file powers search, filters, and article metadata.

1. Create a markdown file

Name it descriptively, like incident-response.md.

2. Add helpful metadata

Include title, description, tags, and a category so the site can organise it.

3. Commit and push

Once the file lands on the default branch, the knowledge base updates automatically.

---
title: Example Article
description: One sentence summary of the article.
category: Operations
tags: [how-to, onboarding]
updated: 2024-02-01
---

Write the rest of your article in Markdown. Use headings, lists, code blocks, and images just like any README.

Need to hide a draft? Add draft: true to the front matter. You can also organise content into sub-folders inside docs/; everything is indexed recursively.

Frequently asked questions

  • Where are the articles stored? Every markdown file lives in the docs/ directory of this repository.
  • How do I share it? Point your custom domain at GitHub Pages or share https://yxuan18.github.io.
  • Can I nest content? Yes. Place markdown files in sub-folders under docs/; they will still be indexed.
  • Can I use another host? Absolutely. Any static hosting service that serves this repository works.
